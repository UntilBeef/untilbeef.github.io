Roblox游戏开发基础

在本教程中，我们将学习Roblox游戏开发的核心概念，包括游戏对象、物理引擎、碰撞检测等。这些概念是构建Roblox游戏的基础，理解它们对于开发高质量的游戏至关重要。

1. 游戏对象（Game Objects）

在Roblox中，所有的游戏元素都是游戏对象（也称为实例，Instance）。就像现实世界中的物体一样，游戏对象可以是角色、道具、地形、光源等。每个游戏对象都有自己的属性和方法。

1.1 常见的游戏对象类型

  • Part：基本的3D立方体，可以用于创建地形、建筑等。
  • Model：用于组织多个游戏对象的容器，就像一个文件夹。
  • Humanoid：玩家角色的控制器，处理移动、跳跃等。
  • Light：光源对象，用于照亮场景。
  • Camera：玩家的视角控制器，决定玩家看到的画面。
  • Script：包含游戏逻辑的脚本，让游戏对象动起来。

3D模型示意图

Roblox中的游戏对象是3D的，它们在三维空间中具有位置、旋转和缩放属性。

1.2 创建和管理游戏对象

创建游戏对象就像在游戏中"建造"东西一样简单！

-- 创建一个新的方块 local part = Instance.new("Part") part.Name = "我的方块" part.Position = Vector3.new(0, 5, 0) -- 位置（x, y, z） part.Size = Vector3.new(2, 2, 2) -- 大小 part.BrickColor = BrickColor.new("Bright blue") -- 颜色 part.Parent = workspace -- 将方块放入游戏世界 -- 组织多个方块到一个模型中 local model = Instance.new("Model") model.Name = "我的模型" local part1 = Instance.new("Part") part1.Name = "方块1" part1.Position = Vector3.new(0, 5, 0) part1.Parent = model local part2 = Instance.new("Part") part2.Name = "方块2" part2.Position = Vector3.new(3, 5, 0) part2.Parent = model model.Parent = workspace

2. 物理引擎

Roblox内置了强大的物理引擎，就像现实世界的物理规则一样，它可以模拟重力、碰撞、摩擦力等效果。

2.1 物理属性

每个游戏对象都有一些物理属性，决定了它在游戏世界中的行为：

  • Mass：对象的质量，影响其惯性和重力响应。质量越大，移动越困难。
  • CanCollide：是否与其他对象碰撞。如果设置为false，物体可以互相穿过。
  • Anchored：是否固定在位置，不受物理影响。固定的物体不会移动或受重力影响。
  • Elasticity：弹性，影响碰撞时的反弹程度。弹性越高，碰撞后反弹越厉害。
  • Friction：摩擦力，影响物体滑动时的阻力。摩擦力越高，物体滑动越困难。

物理引擎工作流程

设置物理属性
物理引擎计算
应用物理效果
更新对象状态

2.2 物理模拟示例

让我们创建一个简单的物理模拟，看看物体是如何受重力影响的：

-- 创建一个受重力影响的方块 local fallingPart = Instance.new("Part") fallingPart.Name = "下落的方块" fallingPart.Position = Vector3.new(0, 20, 0) -- 从高处开始 fallingPart.Size = Vector3.new(2, 2, 2) fallingPart.BrickColor = BrickColor.new("Bright red") fallingPart.Anchored = false -- 不固定，受重力影响 fallingPart.CanCollide = true -- 可以碰撞 fallingPart.Parent = workspace -- 创建一个地面 local ground = Instance.new("Part") ground.Name = "地面" ground.Position = Vector3.new(0, 0, 0) ground.Size = Vector3.new(50, 1, 50) ground.BrickColor = BrickColor.new("Grass green") ground.Anchored = true -- 固定，不受重力影响 ground.Parent = workspace

3. 碰撞检测

碰撞检测就像游戏中的"触觉"，它可以检测游戏对象之间的碰撞，从而触发相应的游戏逻辑。

3.1 碰撞事件

Roblox提供了多种碰撞事件，用于响应不同的碰撞情况：

  • Touched：当对象被其他对象触碰时触发，就像有人碰到了你。
  • TouchEnded：当对象不再被其他对象触碰时触发，就像有人离开了你。
  • TouchMoved：当对象被其他对象触碰并移动时触发，就像有人在你身上移动。

3.2 碰撞检测示例

让我们创建一个触发区域，当玩家进入时显示一条消息：

-- 创建一个可以检测碰撞的区域 local trigger = Instance.new("Part") trigger.Name = "触发区域" trigger.Position = Vector3.new(0, 2, 0) trigger.Size = Vector3.new(5, 1, 5) trigger.BrickColor = BrickColor.new("Bright yellow") trigger.Transparency = 0.5 -- 半透明，让玩家能看到 trigger.CanCollide = false -- 不碰撞，允许玩家穿过 trigger.Anchored = true trigger.Parent = workspace -- 监听碰撞事件 trigger.Touched:Connect(function(hit) -- 检查碰撞的对象是否是玩家 local character = hit.Parent local humanoid = character:FindFirstChild("Humanoid") if humanoid then local player = game:GetService("Players"):GetPlayerFromCharacter(character) if player then print(player.Name .. " 进入了触发区域！") -- 这里可以添加游戏逻辑，如给予奖励、触发事件等 end end end)

4. 坐标系和向量

在Roblox中，游戏世界使用三维坐标系，就像现实世界中的空间一样，每个位置由x、y、z三个坐标组成。向量（Vector3）用于表示位置、方向和速度等。

三维坐标系

Roblox使用右手坐标系：

  • x轴指向右
  • y轴指向上
  • z轴指向前

4.1 基本向量操作

向量就像游戏中的"方向指示器"，可以用来表示位置、方向和速度等：

-- 创建向量 local position = Vector3.new(10, 5, 3) -- x=10, y=5, z=3 local direction = Vector3.new(1, 0, 0) -- 沿x轴正方向（向右） -- 向量运算 local newPosition = position + direction -- 位置偏移（向右移动1单位） local distance = (position - Vector3.new(0, 0, 0)).Magnitude -- 到原点的距离 local normalizedDirection = direction.Unit -- 归一化向量（长度为1） -- 应用向量 local part = workspace:FindFirstChild("MyPart") if part then -- 移动部件 part.Position = part.Position + Vector3.new(0, 1, 0) -- 向上移动1单位 -- 设置部件速度（如果部件不受约束） local bodyVelocity = Instance.new("BodyVelocity") bodyVelocity.Velocity = Vector3.new(10, 0, 0) -- 沿x轴以10的速度移动 bodyVelocity.Parent = part end

5. 游戏循环

游戏循环就像游戏的"心跳"，它持续运行，不断更新游戏状态、处理用户输入、渲染画面等，让游戏保持动态。

游戏循环流程

处理用户输入（如按键、鼠标点击）
更新游戏状态（如位置、生命值）
检测碰撞（如玩家碰到敌人）
渲染画面（显示游戏世界）
重复循环

5.1 使用RunService

Roblox提供了RunService服务，用于创建游戏循环：

-- 获取RunService local RunService = game:GetService("RunService") -- 创建游戏循环函数 local update = function(deltaTime) -- deltaTime是自上一帧以来的时间（秒） -- 这个函数会在每帧执行一次 -- 这里可以添加游戏逻辑，如： -- - 更新游戏状态 -- - 处理用户输入 -- - 移动物体 -- - 检测碰撞 -- - 计算AI行为 end -- 连接到游戏循环 RunService.Heartbeat:Connect(update)

练习 2: 向量操作练习

创建一个可以通过向量控制移动的物体。

  1. 创建一个部件作为玩家控制的物体
  2. 使用UserInputService监听键盘输入
  3. 根据输入方向创建向量
  4. 使用向量控制物体移动
-- 获取UserInputService local UserInputService = game:GetService("UserInputService") -- 获取要移动的部件 local part = workspace:FindFirstChild("Player") -- 移动速度 local moveSpeed = 5 -- 监听键盘输入 UserInputService.InputBegan:Connect(function(input, gameProcessedEvent) if gameProcessedEvent then return end if input.KeyCode == Enum.KeyCode.W then -- 向前移动 part.Position = part.Position + Vector3.new(0, 0, moveSpeed) elseif input.KeyCode == Enum.KeyCode.S then -- 向后移动 part.Position = part.Position + Vector3.new(0, 0, -moveSpeed) elseif input.KeyCode == Enum.KeyCode.A then -- 向左移动 part.Position = part.Position + Vector3.new(-moveSpeed, 0, 0) elseif input.KeyCode == Enum.KeyCode.D then -- 向右移动 part.Position = part.Position + Vector3.new(moveSpeed, 0, 0) end end)

练习：创建一个简单的物理模拟

创建一个场景，包含一个地面和多个从空中落下的物体，当物体触地时改变颜色。

-- 创建地面 local ground = Instance.new("Part") ground.Name = "Ground" ground.Position = Vector3.new(0, 0, 0) ground.Size = Vector3.new(50, 1, 50) ground.BrickColor = BrickColor.new("Grass green") ground.Anchored = true ground.Parent = workspace -- 创建多个下落的物体 local colors = {"Bright red", "Bright blue", "Bright yellow", "Bright green"} for i = 1, 10 do local part = Instance.new("Part") part.Name = "FallingPart" .. i part.Position = Vector3.new( math.random(-20, 20), math.random(10, 30), math.random(-20, 20) ) part.Size = Vector3.new(1, 1, 1) part.BrickColor = BrickColor.new(colors[math.random(1, #colors)]) part.Anchored = false part.CanCollide = true part.Parent = workspace -- 监听碰撞事件 part.Touched:Connect(function(hit) if hit.Name == "Ground" then part.BrickColor = BrickColor.new("Black") part.Anchored = true end end) end
